# صفحة «غير موجود» (404) — Not Found

توثيق مختصر لمكوّن تجربة الصفحة غير الموجودة في الواجهة الأمامية.

## الملفات

| الملف | الدور |
|--------|--------|
| `app/not-found.tsx` | نقطة دخول Next.js لعرض واجهة 404؛ يعرض الهيدر ثم التجربة. |
| `components/not-found/NotFoundExperience.tsx` | منطق العميل: حركة، صوت، لغة، روابط. |
| `components/not-found/NotFoundExperience.module.css` | الأنماط والألوان (هوية أناناس: أصفر، وردي، أخضر، خلفية فاتحة). |
| `lib/translations.ts` | مفاتيح `notFound.*` بالعربية والإنجليزية. |

## اللغة (عربي / إنجليزي)

- تُستخرج اللغة من **أول جزء في المسار**: `/ar/...` أو `/en/...` (بما يتوافق مع الـ middleware).
- إن لم تُعرف لغة مدعومة، يُستخدم الافتراضي من `lib/localization.ts` (`DEFAULT_LOCALE`).
- النصوص تُعرض عبر `t('notFound.…', locale)`.

### مفاتيح الترجمة

- `notFound.badge`
- `notFound.title`
- `notFound.lead`
- `notFound.backHome`
- `notFound.siteName`

لتعديل النصوص: عدّل القيم في `translations.ts` ضمن كائنَي `ar` و `en`.

## الحركة (Motion)

- المكتبة: **`motion/react`** (حزمة `motion`).
- عند تفعيل **«تقليل الحركة»** في النظام (`prefers-reduced-motion`) تُعطّل الحركات اللانهائية وتُلغى محاولة تشغيل الصوت.

## الصوت

- تنبيه قصير عبر **Web Audio API** (بدون ملفات صوتية).
- يُشغَّل مرة بعد تأخير بسيط عند التحميل؛ قد يمنع المتصفح التشغيل التلقائي حتى يحدث تفاعل من المستخدم.

## الألوان والواجهة

- الهوية البصرية متماشية مع الصفحة الرئيسية: **أصفر** للتمييز، **وردي** لزر العودة، **أخضر** كلون مساعد، **خلفية فاتحة** وبطاقة بيضاء بزوايا دائرية كبيرة.
- لا يُستخدم الأزرق كلون أساسي في هذه الصفحة.

## الرابط إلى الرئيسية

- الزر يوجّه إلى `/${locale}` للحفاظ على لغة التصفح الحالية.

## إمكانية الوصول

- `<main lang={locale}>` لمساعدة قارئات الشاشة.
- حالات التركيز على الزر تستخدم `focus-visible` مع إطار واضح.

## تطوير لاحق (اختياري)

- ربط ألوان CSS بمتغيرات عامة في SCSS إن وُجدت لوحة موحّدة.
- إضافة ملف صوتي اختياري إذا رُغب في تنبيه ثابت بدل Web Audio.
