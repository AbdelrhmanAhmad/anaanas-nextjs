# 11 — قائمة فحص يدوي (QA Checklist)

استخدم هذه القائمة قبل دفع التغييرات إلى الـ staging.

## أ. الـ Layout والترتيب

- [ ] **ديسكتوب (`>= lg`)**: `Banner → CreatePost → Feeds` بدون ظهور
      الصف الخاص بالموبايل.
- [ ] **موبايل (`< md`)**: `Banner → CreatePost → DiscoveryRow → Feeds`.
- [ ] لا يوجد نص حرفي يبدأ بـ `//` ظاهر على الصفحة (تعليقات JSX سليمة).

## ب. الرأس وبانر التحديثات

- [ ] الشعار بحجم صحيح (80×34px) على الموبايل بدون قفز عند التحميل.
- [ ] زر الإشعارات يعرض العدّاد الصحيح ويفتح القائمة المنسدلة.
- [ ] زر الرسائل يعرض عدد غير المقروء من `ChatContext`.
- [ ] شريط البحث: ميكروفون + زر مسح يعملان.
- [ ] Quick actions في `MobileDailyUpdatesBanner` تظهر **فقط** لمستخدم
      `28775`.

## ج. الشبكات

- [ ] `MobileSectionsSwiper`: 4 أعمدة × 2 صفوف + بلاطة "عرض الكل".
- [ ] المودال يفتح ويغلق بـ TAB/ESC.
- [ ] لا توجد فجوة كبيرة (padding) قبل الشبكة بسبب `_reboot.scss`.

## د. `MobileAIDashboard`

- [ ] دخول البطاقات الست **بالتتابع** (stagger).
- [ ] Hover على كل بطاقة يرفعها ويظهر ظل ملوّن.
- [ ] الأيقونات SVG بحدود واضحة، لا صور بكسلية.
- [ ] شارة Live تنبض، نقطة خضراء واضحة.
- [ ] العنوان متدرّج وملوّن.

## هـ. `MobileAiPromoGrid`

- [ ] الصف العلوي 3 بطاقات معلوماتية بدون تمدد ارتفاع.
- [ ] صف Hot deals + Auctions بعرضين متساويين.
- [ ] العداد التنازلي يعمل بـ `aria-live="polite"`.
- [ ] شريط Smart ads العريض يستخدم SVG (لا صورة أناناس).

## و. الحركة (Motion)

- [ ] فعّل **Reduce motion** في إعدادات النظام:
  - [ ] لا أنيميشن لانهائي.
  - [ ] لا hover transform.
  - [ ] القسم يدخل فوراً بدون stagger.

## ز. RTL / LTR

- [ ] التبديل بين `/ar` و `/en` يظهر الترتيب الصحيح للحقول.
- [ ] الأيقونات (سهم/مطرقة) في المكان المناسب RTL.
- [ ] `inset-inline-start/end` تعمل بشكل صحيح.

## ح. شاشات صغيرة

- [ ] `320px` و `360px` بدون scroll أفقي.
- [ ] لا يقطع نص عن البطاقة (`ellipsis` / `line-clamp` فعّالان).

## ط. Backend / API

- [ ] `/api/posts?country_id=…&land=ar` يستجيب بـ `200`.
- [ ] لا أخطاء `Unknown column 'post_type'` أو `'publish_date'` في
      `storage/logs/laravel.log`.
- [ ] بعد `php artisan migrate`:
  - [ ] ترتيب الـ feed يستخدم `publish_date` عند توفره.
  - [ ] فلتر `post_type = 'listing'` يُطبق.

## ي. أخطاء معروفة جانبية

- [ ] إذا كانت Mongo غير شغّالة: لا يجب أن تفشل الصفحة كليّاً (fail-safe).
- [ ] إذا كانت Mongo شغّالة: العدّ الفعلي للـ reactions يظهر على البطاقات.
